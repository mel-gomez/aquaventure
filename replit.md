# Aquaventure Swim School

## Overview

Full-stack web application for Aquaventure Swim School (est. 2019) ‚Äî a children-focused swim school in Angono, Rizal, Philippines. Includes a public landing page, parent/swimmer portal, and admin dashboard.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Frontend**: React + Vite (wouter routing), Tailwind CSS, Framer Motion, Recharts
- **Backend**: Express.js v5
- **Database**: PostgreSQL + Drizzle ORM
- **Authentication**: JWT (access + refresh tokens via jsonwebtoken + bcryptjs)
- **Validation**: Zod (server routes use `"zod"` directly, NOT `"zod/v4"`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec in `lib/api-spec/openapi.yaml`)
- **Build**: esbuild

## Key Commands

- `pnpm run typecheck` ‚Äî full typecheck across all packages
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API hooks + Zod schemas
- `pnpm --filter @workspace/db run push` ‚Äî push DB schema changes (dev only)
- `pnpm --filter @workspace/scripts run seed-new-tables` ‚Äî seed testimonials + FAQ

## App Structure

### Public Frontend (`artifacts/aquaventure`)
- `/` ‚Äî Landing page: hero, features, programs, local identity, testimonials (from DB), CTA
- `/programs` ‚Äî Browse all swim programs with pricing
- `/schedule` ‚Äî Class schedule with level filters
- `/about` ‚Äî School story, coaches, mission
- `/faq` ‚Äî Accordion FAQ from DB (category filters)
- `/contact` ‚Äî Contact form + address/phone/email

### Parent Portal (protected)
- `/portal` ‚Äî Dashboard: active enrollments + cancel button, announcements
- `/portal/enroll` ‚Äî Enrollment form (pick session, enter swimmer details)
- `/portal/attendance` ‚Äî Swimmer attendance history + rate
- `/portal/profile` ‚Äî Edit name, phone; view role + email
- `/portal/progress` ‚Äî View swimmer skill milestones logged by coaches

### Admin Dashboard (protected, admin-only)
- `/admin` ‚Äî Stats cards, enrollment bar chart, recent enrollments
- `/admin/swimmers` ‚Äî User directory
- `/admin/enrollments` ‚Äî Confirm/cancel enrollments
- `/admin/programs` ‚Äî Create programs
- `/admin/sessions` ‚Äî Create sessions
- `/admin/announcements` ‚Äî Post announcements
- `/admin/attendance` ‚Äî Mark student attendance (present/absent/excused)
- `/admin/progress` ‚Äî Log/delete swimmer skill achievements (19 preset skills + custom)
- `/admin/testimonials` ‚Äî Approve/hide/delete parent testimonials
- `/admin/faq` ‚Äî Add/delete FAQ entries
- `/admin/contacts` ‚Äî View contact form submissions, mark read/replied

## Architecture Decisions
- Server routes import `zod` (not `zod/v4`) ‚Äî esbuild can't resolve the `/v4` subpath
- API client hooks generated by Orval into `lib/api-client-react/src/generated/`
- `zod` added as explicit dep to `@workspace/api-server` for inline validation in new routes
- FloatingEnrollButton hides on all `/portal*` and `/admin*` paths via `startsWith`

## Design
- Ocean gradient palette: deep navy (#0a1628) ‚Üí aqua (#00c4cc), accent `#00c4cc`
- Glassmorphism card accents, CSS wave SVG pattern in hero sections
- Framer Motion scroll animations throughout
- Aquaventure Giants logo: `attached_assets/image_1777978704079.png` via `@assets` alias ‚Äî never apply `brightness-0 invert`

## Gotchas
- Logo in admin sidebar and hero: do NOT apply `brightness-0 invert` (it's already colored)
- `pricePerSession` stored as `numeric` in DB ‚Äî use `parseFloat()` when reading in routes
- Never import from `"zod/v4"` in server code ‚Äî use `"zod"` only
